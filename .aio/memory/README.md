# Memory Directory

このディレクトリには、Planner が学習した知識をカテゴリ別に保存します。

## 目的

- プロジェクトに関する知識を永続化する
- 同じ質問を繰り返さない
- コンテキストを保持する
- 意思決定の履歴を残す

## カテゴリ

| ファイル | 内容 |
|---------|------|
| `architecture.md` | プロジェクトのアーキテクチャに関する知識 |
| `tech-stack.md` | 使用している技術スタックの情報 |
| `requirements.md` | プロジェクトの要件や仕様 |
| `decisions.md` | 過去の意思決定とその理由 |
| `conventions.md` | コーディング規約やプロジェクトの規約 |

## 使い方

Planner は以下のタイミングでメモリを利用します：

1. **読み込み**: タスクを受け取った際に関連する知識を参照
2. **保存**: 新しい情報を学習したときや意思決定をしたときに記録
3. **更新**: 既存の知識が古くなったり、変更があった場合に更新

## 注意事項

- 各ファイルは Markdown 形式で記述
- 機密情報は保存しない
- 定期的に内容を見直し、最新の状態に保つ
